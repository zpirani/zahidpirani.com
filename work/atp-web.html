Context

ATP launched mobile-first in 2021. For more than two years, mobile was the only ATP surface; every interaction model and design pattern was proven there first.

ATP Web launched in June 2023 to address what mobile genuinely couldn't do: deep technical analysis, larger data views, and multi-screen workflows for active traders. I led the design from launch through September 2024 — 15 months of dedicated web work alongside ongoing mobile responsibilities.

The central decision: web would extend mobile, not replace it. Web inherited ATP's design language, interaction patterns, IA, and visual identity — then added what desktop's screen real estate, input model, and continuous attention made possible.

The challenge

Three problems shaped every web design decision:

  1. Honour the mobile-first identity. Mobile had carried ATP for two years. Existing traders had patterns burned into their hands. Web couldn't feel like a different product.
  2. Deliver the professional-grade tools desktop traders actually need. Advanced charting, larger data views, multi-screen workflows — the things mobile couldn't deliver.
  3. Avoid feature-parity-as-translation traps. A naive port of every mobile screen would produce a desktop product that felt like a tablet app. Each surface had to be reconsidered for what desktop could actually deliver — even when the underlying flow was the same.

What I inherited from Mobile

Web didn't start from zero. Mobile had already established:

  • The Axi Design System (built in parallel with mobile, governing colour, type, components, motion)
  • Information architecture for the core trading product (markets → order → position → history)
  • Onboarding flow and jurisdiction logic (FCA, CySEC, ASIC, DFSA, FMA, W-8BEN for US, conditional Jumio for high-risk)
  • Visual identity and tone of voice
  • The mobile cohort metrics (−35% setup, 2× conversion) that proved the patterns worked

The job on Web was to carry these forward, not relitigate them.

The three depth additions

1. Main Trading Interface

The core trading workspace — where traders browse markets, analyse instruments, and place trades. The two ingredients that made web's interface distinct were a more visible markets view and a responsive order ticket that lived alongside the chart, rather than appearing as a modal as it does on mobile.

TradingView integration sat at the heart of this interface — the industry-standard charting platform, integrated so traders had access to professional-grade indicators, drawing tools, and timeframes without leaving Axi.

2. Portfolio

Mobile's position management was inevitably constrained — small screen, limited filters, summary views. On Web, I introduced Portfolio as a central hub: open positions, balances, and overall performance in one view, with clear hierarchy and quick actions (close, modify) inline.

The design emphasised making complex financial data approachable — visible by default but not overwhelming, with the depth tucked behind drill-downs and inline action surfaces.

3. In-Flow Funding

When a trader attempts to execute a trade but lacks sufficient funds, mobile and most competitor platforms surface a dead-end: "insufficient funds, please deposit." On Web, I designed an inline funding dialog that presents the user with options to use a saved payment method or choose an alternative funding method — embedded inside the trade flow, not separated from it.

The design preserves the trader's momentum: they can fund and complete the trade without losing context, watchlists, or the order they were placing. It removes the most common abandonment point in any mid-trade interaction.

Outcomes

ATP Web was instrumented in Amplitude, but I retain only the mobile cohort data — the same caveat from the Mobile case study. Web outcomes here are framed qualitatively.

  • Launched ATP Web in June 2023 as a feature-parity desktop surface from day one — no extended beta or feature gap that mobile users couldn't recognise.
  • Inherited and preserved the mobile-cohort onboarding metrics (−35% setup, 2× conversion) through to the web flow.
  • Extended the Axi Design System across two surfaces under one design language.
  • Pioneered an in-flow funding pattern that removed mid-trade abandonment cliffs — a specific design call, not a port from mobile.
